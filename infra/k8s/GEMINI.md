# Kubernetes Agents Guide

## Stack
- Helmfile
- Helm v3

## Commands
```bash
task k8s:diff K8S_ENV=<env>    # Preview changes
task k8s:apply K8S_ENV=<env>   # Deploy releases
task k8s:destroy K8S_ENV=<env> # Teardown releases
```

## Hard Limits
- **NEVER** hardcode credentials in values files. You must use the `charts/secrets` deployment pattern for injecting SOPS data as Kubernetes Secrets.

## Conventions
- **`.gotmpl` Extensions:** Values files use `.yaml.gotmpl` because they are pre-processed by Helmfile for Go templating (e.g., reading external JSON files or environment variables).
- **Unified App Chart:** `charts/harmony` is a generic chart used to deploy both the FastAPI backend and the Python CDC consumer by supplying different value configurations.
- **Secrets Hub:** All secrets are centrally rendered into Kubernetes `Secret` resources by the `harmony/secrets` release. Other workloads map these secrets via `secretEnv` configurations.

## Gotchas
- **Missing `tf-outputs.json`:** Helmfile overrides in staging/production (`values-override.yaml.gotmpl`) will crash if `../../../terraform/environments/<env>/tf-outputs.json` hasn't been generated by Terraform first.
- **Local vs Remote Asymmetry:** `dynamodb-local` and `init-job` releases only run locally. Karpenter (node autoscaling) and `gp3-storage-class` only run remotely.

## Structure Pointers
- `helmfile.yaml.gotmpl`: The master orchestrator defining release order and dependencies.
- `environments/base/values/`: The default configurations for all charts. Look here before overriding in specific environments.
